TCSS 305 Game Application

Educational demonstration of modern Java design patterns.

This is a simple grid-based game that demonstrates best practices in Java application design, including MVC architecture, the Observer pattern, and modern Java language features.

Application Entry Point

Main Class: {@link edu.uw.tcss.game.Application#main(String[])}

Package Structure

{@link edu.uw.tcss.game.model}
Game logic and state management - Implements the Model in MVC pattern with Singleton and Observer patterns
{@link edu.uw.tcss.game.gui.controller}
User input handling and game control - The Controller in MVC pattern
{@link edu.uw.tcss.game.gui.view}
Visual representation of game state - The View in MVC pattern
{@link edu.uw.tcss.game.gui.util}
GUI utility classes for icon creation and visual assets

Key Design Patterns

  • Model-View-Controller (MVC) - Separation of concerns across packages
  • Singleton - Single game instance pattern
  • Observer - PropertyChangeListener for event-driven updates
  • Immutable Records - Type-safe data transfer objects
  • Sealed Classes - Controlled type hierarchies for events

Modern Java Features

This project demonstrates Java 21 features including records, sealed classes, pattern matching in switch expressions, and enhanced instanceof.
